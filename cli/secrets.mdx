## Overview

These commands let you manage individual secrets in your vault. All secrets support versioning and categorization.

## Creating & Updating Secrets

### set

Create a new secret or update an existing one.

```bash
imitsu set <name> [value]
```

<ParamField path="name" type="string" required>
  The unique name for the secret (e.g., DATABASE_URL, API_KEY)
</ParamField>

<ParamField path="value" type="string">
  The secret value. If omitted, you'll be prompted to enter it securely
</ParamField>

<ParamField path="-c, --category" type="string" default="general">
  Secret category for organization (e.g., database, api, auth)
</ParamField>

<ParamField path="-t, --team" type="string">
  Share with team by name (grants read access)
</ParamField>

**Example (with value):**
```bash
imitsu set DATABASE_URL "postgresql://localhost/mydb"
```

**Output:**
```text
Created: DATABASE_URL
```

**Example (prompted for value):**
```bash
imitsu set API_KEY -c api
```

**Interactive prompt:**
```text
Secret value: 
```

**Output:**
```text
Created: API_KEY
```

**Example (with team sharing):**
```bash
imitsu set STRIPE_KEY "sk_test_123" -c payment -t engineering
```

**Output:**
```text
Created: STRIPE_KEY
Shared with team "engineering" (read access)
```

**Output:**
```text
Created: STRIPE_KEY
Shared with team "engineering" (read access)
```

**Updating existing secrets:**
If a secret with the same name exists, it will be updated:
```text
Updated: DATABASE_URL
```

## Retrieving Secrets

### get

Retrieve a secret's value by name.

```bash
imitsu get <name>
```

<ParamField path="name" type="string" required>
  The name of the secret to retrieve
</ParamField>

The command outputs only the secret value (no extra text), making it perfect for piping to other commands or scripts.

**Example:**
```bash
imitsu get DATABASE_URL
```

**Output:**
```text
postgresql://localhost/mydb
```

**Example (using in scripts):**
```bash
export DATABASE_URL=$(imitsu get DATABASE_URL)
echo $DATABASE_URL
```

**Error (secret not found):**
```
Secret not found: MISSING_KEY
```

## Listing Secrets

### list (ls)

List all secrets you have access to.

```bash
imitsu list
imitsu ls
```

<ParamField path="-m, --mine" type="boolean" default="false">
  Show only secrets you created (excludes shared secrets)
</ParamField>

**Example:**
```bash
imitsu ls
```

**Output:**
```text
NAME                          CATEGORY       VER   UPDATED
──────────────────────────────────────────────────────────────────────
DATABASE_URL                  general        v1    2026-03-06T10:30:00Z
API_KEY                       api            v2    2026-03-06T11:15:00Z
STRIPE_KEY                    payment        v1    2026-03-06T11:45:00Z
```

**Example (only your secrets):**
```bash
imitsu ls --mine
```

**Output (when no secrets exist):**
```text
No secrets found
```

## Deleting Secrets

### rm

Delete a secret permanently.

```bash
imitsu rm <name>
```

<ParamField path="name" type="string" required>
  The name of the secret to delete
</ParamField>

**Example:**
```bash
imitsu rm OLD_API_KEY
```

**Output:**
```text
Deleted: OLD_API_KEY
```

**Error (secret not found):**
```text
Secret not found: MISSING_KEY
```

## Categories

Categories help organize secrets by purpose:

- `general` (default) - Miscellaneous secrets
- `database` - Database connection strings
- `api` - API keys and tokens
- `auth` - Authentication credentials
- `payment` - Payment gateway keys

You can use any category name that makes sense for your team. Categories are especially useful when exporting secrets.
