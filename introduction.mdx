## What is imitsu?

imitsu is a secure, self-hosted team secret manager designed for development teams who need to share API keys, credentials, and environment variables safely. Built with security at its core, imitsu provides enterprise-grade encryption while maintaining simplicity in deployment and usage.

## Key Features

<CardGroup cols={2}>
  <Card title="Military-Grade Encryption" icon="lock">
    AES-256-GCM encryption with per-secret salts, IVs, and keys derived via HKDF-SHA512
  </Card>
  <Card title="Team Collaboration" icon="users">
    Share secrets with individuals or teams with granular read/write/admin permissions
  </Card>
  <Card title="Comprehensive Audit Trail" icon="clipboard-list">

  </Card>
  <Card title="Multiple Interfaces" icon="terminal">

  </Card>
  <Card title="Bulk Operations" icon="file-import">

  <Card title="Simple Deployment" icon="server">

  </Card>
</CardGroup>

## Security First

imitsu is built with security as the top priority:

- **Encryption**: AES-256-GCM with per-secret salts and initialization vectors
- **Password Hashing**: Argon2id with 65MB memory cost and 3 iterations
- **Authentication**: JWT tokens with 8-hour expiry
- **Access Control**: Role-based permissions (admin/member) and per-secret sharing
- **Rate Limiting**: 100 requests per minute per IP address
- **Audit Logging**: Complete audit trail of all operations

<Note>
  imitsu stores all secrets encrypted at rest. The master encryption key never leaves your server, and secrets are only decrypted when explicitly requested by authorized users.
</Note>

## Architecture

imitsu consists of three main components:

1. **API Server** (Express + TypeScript) - RESTful API with JWT authentication
2. **CLI Client** (Node.js + Commander) - Command-line interface for secret management
3. **TUI Client** (Go + Bubble Tea) - Interactive terminal UI for visual workflows

All components share the same configuration and can be used interchangeably based on your workflow preferences.
