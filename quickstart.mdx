## CLI Quickstart

<Steps>

<Step title="Start the Server">

First, clone the repository and start the imitsu server:

```bash
git clone https://github.com/adeleke5140/imitsu.git
cd imitsu
npm install
npm run build
```

Start the server with required environment variables:

```bash
IMITSU_MASTER_KEY="your-secret-master-key-min-32-chars" \
IMITSU_JWT_SECRET="your-jwt-secret-min-32-chars" \
npm start
```

<Warning>
  In production, use strong random values for `IMITSU_MASTER_KEY` and `IMITSU_JWT_SECRET`. Never commit these to version control.
</Warning>

You should see:

```
imitsu server running on port 3100
Health check: http://localhost:3100/health
```

The server defaults to `http://localhost:3100`. You can change the port with `IMITSU_PORT`.

</Step>

<Step title="Register Your Account">

In a new terminal, register as the first user (you'll automatically become admin):

```bash
imitsu register -e you@team.com -n "Your Name"
```

You'll be prompted to enter a password:

```
Password (min 12 chars): 
Confirm password: 
Registered! Role: admin
You are the first user — you've been made admin.
```

<Note>
  The first user to register automatically receives the `admin` role. Subsequent users get the `user` role by default.
</Note>

</Step>

<Step title="Login">

Authenticate with your new account:

```bash
imitsu login you@team.com
```

Enter your password:

```
Password: 
Logged in as you@team.com (admin)
```

Your authentication token is saved to `~/.imitsu/config.json` for subsequent commands.

</Step>

<Step title="Create Your First Secret">

Store a secret value:

```bash
imitsu set DATABASE_URL "postgresql://user:pass@localhost:5432/mydb"
```

Output:

```
Created: DATABASE_URL
```

You can also create secrets interactively (useful for passwords):

```bash
imitsu set API_KEY
```

You'll be prompted securely:

```
Secret value: 
Created: API_KEY
```

</Step>

<Step title="Retrieve Your Secret">

Fetch the secret value:

```bash
imitsu get DATABASE_URL
```

Output:

```
postgresql://user:pass@localhost:5432/mydb
```

The output is pipeable, making it perfect for scripts:

```bash
export DATABASE_URL=$(imitsu get DATABASE_URL)
```

</Step>

<Step title="List All Secrets">

See all secrets you have access to:

```bash
imitsu ls
```

Output:

```
NAME                          CATEGORY       VER   UPDATED
──────────────────────────────────────────────────────────────────────
DATABASE_URL                  general        v1    2026-03-06T10:30:15Z
API_KEY                       general        v1    2026-03-06T10:31:42Z
```

</Step>

</Steps>

## TUI Quickstart

<Steps>

<Step title="Start the Server">

If you haven't already, start the imitsu server:

```bash
git clone https://github.com/adeleke5140/imitsu.git
cd imitsu
npm install
npm run build

IMITSU_MASTER_KEY="your-secret-master-key-min-32-chars" \
IMITSU_JWT_SECRET="your-jwt-secret-min-32-chars" \
npm start
```

Wait for:

```
imitsu server running on port 3100
Health check: http://localhost:3100/health
```

</Step>

<Step title="Launch the TUI">

Open a new terminal and start the interactive UI:

```bash
itui
```

You'll see the imitsu terminal interface with three main tabs:

- **s** - Secrets (list, view, create, delete, export)
- **t** - Teams (list teams, view members)
- **a** - Account (login, register, server config, profile)

On first launch, you'll land on the **Account** tab.

</Step>

<Step title="Configure Server URL">

Use the arrow keys to navigate to **server** in the sidebar and press `Enter`.

Enter the server URL:

```
http://localhost:3100
```

Press `Enter` to save. The TUI will validate the connection.

</Step>

<Step title="Register Your Account">

Navigate to **register** in the Account sidebar and press `Enter`.

Fill in the form:
- Email: `you@team.com`
- Name: `Your Name`
- Password: (minimum 12 characters)

Press `Tab` to move between fields. Press `Enter` to submit.

You'll see a confirmation:

```
Registered! Role: admin
You are the first user — you've been made admin.
```

</Step>

<Step title="Create a Secret">

The TUI will automatically switch to the **Secrets** tab after login.

Press `n` to create a new secret.

Fill in the form:
- Name: `DATABASE_URL`
- Value: `postgresql://user:pass@localhost:5432/mydb`
- Category: `general` (default)

Press `Enter` to save.

</Step>

<Step title="View Your Secret">

You'll see your new secret in the list:

```
NAME              CATEGORY    VER
──────────────────────────────────
DATABASE_URL      general     v1
```

Use `j` and `k` to navigate up and down, then press `Enter` to view details.

In the detail view:
- Press `d` to delete
- Press `esc` to go back to the list

</Step>

</Steps>

## Common Next Steps

### Import Existing Secrets

If you have a `.env` file, import it in bulk:

```bash
imitsu import .env.production -c production
```

Section headers in your `.env` file become categories automatically:

```bash
### Database ###
DB_HOST=localhost
DB_PASSWORD=secret123

### API Keys ###
STRIPE_KEY=sk_test_...
SENDGRID_KEY=SG...
```

### Export for Local Development

Export secrets back to a `.env` file:

```bash
imitsu export .env.local
```

Or pipe directly to your environment:

```bash
eval $(imitsu export | sed 's/^/export /')
```

### Set Up Team Access

```bash
# Create a team
imitsu team create backend

# Add members
imitsu team add backend dev@team.com

# Share secrets with the team
imitsu share-team DATABASE_URL backend -p read
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Connection refused error">
    Make sure the server is running on the correct port:
    
    ```bash
    # Check health endpoint
    curl http://localhost:3100/health
    ```
    
    If you changed the port, update your client configuration:
    
    ```bash
    imitsu server http://localhost:YOUR_PORT
    ```
  </Accordion>
  
  <Accordion title="Authentication errors">
    Your token may have expired (8-hour expiry). Login again:
    
    ```bash
    imitsu login your@email.com
    ```
  </Accordion>
  
  <Accordion title="Password requirements">
    Passwords must be at least 12 characters. imitsu uses Argon2id hashing with:
    - 65MB memory cost
    - 3 iterations
    - Parallelism: 4
  </Accordion>
</AccordionGroup>
